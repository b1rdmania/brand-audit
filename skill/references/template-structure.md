# Audit Report — HTML Template Reference

Reports are generated by `scripts/generate-report.mjs` from `audit-data.json`. The generator produces a self-contained HTML file — all CSS inline, no external dependencies.

## How to Generate

```bash
PROJECT="/Users/andy/Cursor Projects 2026/brand-audit-tool"
node "${PROJECT}/scripts/generate-report.mjs" "${PROJECT}/data/${SLUG}/audit-data.json"
# Output: reports/{slug}/audit-report.html
```

To override the output path:
```bash
node "${PROJECT}/scripts/generate-report.mjs" input.json /path/to/output.html
```

## Design System

```css
--bg-primary: #faf9f7;      /* cream background */
--bg-white: #ffffff;         /* card background */
--text-primary: #131314;     /* charcoal — headings, strong text */
--text-secondary: #6b6b6f;  /* body text */
--text-tertiary: #9b9b9f;   /* meta text, table headers */
--accent: #d97757;           /* terracotta — links, callouts, priority numbers */
--accent-light: #f5ebe6;    /* callout background */
--green: #2d8a4e;            /* active/good status */
--red: #c4442a;              /* missing/broken status */
--yellow: #b8860b;           /* stale/warning status */
--border: #e8e8e5;
--radius: 10px;
--radius-lg: 14px;
```

## Report Sections

The generator renders sections from the JSON. Content should be shaped by what the discovery found — not forced into a template.

1. **Header** — `meta.business_name`, `meta.generated`, subtitle from `executive_summary.core_finding`
2. **Executive Summary** — callout blocks for `core_finding` and `opportunity`, then `business_context`
3. **Scorecard** — grid from `findings` array (each item's `category` + `score`)
4. **Website Overview** — cards from `website` object (platform, PageSpeed, SSL, blog)
5. **Presence Inventory** — table from `presence` array (channel, url, handle, status, notes)
6. **Detailed Findings** — card per item in `findings` array (category, score, summary, evidence[])
7. **Prioritised Actions** — from `actions` object (this_week[], this_month[], ninety_days[])
8. **Content Strategy** — card grid from `content_strategy` array (title, description)
9. **Existing Strengths** — bullet list from `strengths` array (supports HTML in strings)
10. **Next Steps** — Claude prompt from `claude_prompt.prompt_text` with copy-to-clipboard button
11. **Footer** — date and disclaimer

## JSON → HTML Mapping

The generator supports HTML in content fields (findings summaries, evidence, strengths). Use `<strong>`, `<a href>`, etc. directly in the JSON strings — they render as-is.

The `meta.slug` field determines the output directory. If missing, it's derived from `meta.business_name`.
